01

단답형 암기 코어

1–12번 · 각 3점 · 36점

여기는 암기형이 대부분. 용어·수치·명령을 정확히 외우면 응용 없이도 득점됩니다. 다만 ⚠︎ 표시(iptables·로그경로)는 "규칙을 직접 채우는" 형태로 변형되니 손으로 써보세요.

shadow / passwd 구조암기29·30·31 출제
🔑 기출 핵심
$id$salt$hash — id(1=MD5·5=SHA256·6=SHA512), salt=솔트, hash=해시값. shadow가 활성화되면 /etc/passwd의 2번째 필드는 x.
$id$salt$hashxsolt=레인보우테이블 방어
🔄 응용 포인트
passwd 필드값(x)·pwconv/pwunconv·솔트가 레인보우테이블을 막는 이유까지 물음.
Q. 솔트(salt)가 레인보우테이블 공격을 막는 이유는?
해시 연산 시 salt를 추가 입력으로 쓰면 같은 비밀번호라도 매번 해시값이 달라져, 미리 계산해 둔 해시-평문 대응표(레인보우테이블)를 무력화한다.
+ pwunconv: shadow의 해시를 passwd로 되돌리고 shadow 비활성화.
iptables 규칙 ⚠︎변형응용23·24·31 출제
🔑 기출 핵심
외부→내부 ping 차단: iptables -A INPUT -p icmp --icmp-type 8 -j DROP (icmp-type 8=echo-request, 0=echo-reply)
🔄 응용 포인트
체인(INPUT/OUTPUT/FORWARD)·프로토콜(-p tcp --dport)·타깃(DROP/REJECT/ACCEPT)을 바꿔 빈칸으로 냄. 규칙을 직접 완성할 수 있어야 함.
Q. 내부→외부로 나가는 80·443 응답만 허용하는 규칙 골격은?
iptables -A OUTPUT -p tcp -m multiport --dport 80,443 -j ACCEPT · 리턴 트래픽은 -m state --state ESTABLISHED,RELATED -j ACCEPT.
암기 포인트: -A(추가)/-p(프로토콜)/--dport(목적지 포트)/-j(타깃) 4요소.
DNS 스푸핑 / 캐시 포이즈닝암기22·26·29·31 출제
🔑 기출 핵심
정상 DNS보다 빠른 위조 응답으로 가짜 사이트 유도(=DNS 스푸핑, MITM 일종). 캐시 포이즈닝=DNS 캐시를 오염.
🔄 응용 포인트
대응책(DNSSEC·캐시 TTL 관리·랜덤 소스포트/쿼리ID)과 sniffing 기반 선행 조건을 물을 수 있음.
보안 도구 명칭 세트암기23·25·28·31 출제
🔑 기출 핵심
  • Tripwire = 파일 무결성 점검 도구
  • Nessus(Tenable) = 취약점 진단 스캐너
  • SIEM = 로그 빅데이터 상관분석·포렌식·모니터링(← ESM 진화형)
  • Fuzzing(퍼징) = 무작위 데이터 입력으로 취약점 탐지
🔄 응용 포인트
SIEM↔ESM↔SOAR 차이, IDS/IPS·SOAR와의 연계 등 "무엇의 진화/차이냐"로 변형.
윈도우 로그 경로 ⚠︎변형응용23·29·31 출제
🔑 기출 핵심
IIS 로그 C:\Windows\inetpub\logs\LogFiles\W3SVC1, HTTPERR·DHCP 로그 경로. 이벤트 로그=이벤트 뷰어.
🔄 응용 포인트
29-17처럼 최대 로그 크기 계산(단일이벤트크기 × 일일건수 × 보관일)로 변형. 계산식·단위(byte→KB) 주의.
Q. 단일 500B · 하루 1,000건 · 30일 보관이면 최대 로그 크기는?
500 × 1,000 × 30 = 15,000,000 바이트 ≈ 15,000 KB(≈14.3 MB). 계산식을 반드시 함께 써야 부분점수 방어.
위험관리 절차·접근법암기다수 회차 단골
🔑 기출 핵심
절차: 전략수립 → 위험분석 → 위험평가 → 대책선정 → 계획수립 → 구현·운영.
3대 접근법: 기준선 / 상세 / 복합(고위험=상세, 그 외=기준선).
🔄 응용 포인트
ISO 27005 용어(위험식별/분석/수준평가)로 치환, 위험처리 4종(수용·감소·전가·회피)과 엮어 서술형으로 확장.
법령 정의 용어암기다수 · 31-11
🔑 기출 핵심
정보통신망 / 정보보호시스템 / 정보통신기반시설 / 내부관리계획 / 전자적 침해행위 / 집적정보통신시설 사업자(IDC, 망법 §46) 등 법조문 정의.
🔄 응용 포인트
접속기록 보관(1년/2년·5만명↑=2년), 개인정보 안전성 확보조치 수치와 함께 빈칸형으로.
02

서술형 개념 + 응용

13–16번 · 각 12점 · 48점

배점의 절반. 정의만 쓰면 절반 점수, 원리·단계·보안적 의의까지 재구성해야 만점입니다. 각 카드의 자가점검은 "다른 각도로 물었을 때"를 연습하는 문항입니다.

SSL/TLS 핸드셰이크응용22·24·26·29·31 출제
🔑 기출 핵심
Client Hello ↔ Server Hello로 암호 스위트·난수 교환 → 서버 인증서 검증 → 세션키(pre-master) 교환.
🔄 응용 포인트 (31-16 실제 출제 각도)
  • 비대칭키(공개키) = 인증서 검증·전자서명 확인·세션키 교환 단계에 사용
  • 대칭키(세션키) = 이후 본 통신 데이터 암호화에 사용(성능 때문에)
"어느 단계에서 어떤 키를 왜 쓰나"를 반드시 연결해서 서술.
Q. 왜 비대칭키로 끝까지 통신하지 않고 대칭키로 전환하는가?
비대칭(공개키) 연산은 연산량이 크고 느림 → 초기 인증·키 교환에만 쓰고, 실제 대량 데이터는 빠른 대칭키로 암호화한다. 하이브리드 암호화의 핵심.
XSS 유형·대응응용26·31 출제
🔑 기출 핵심
  • 저장형(Stored) — 서버 DB에 스크립트 저장, 열람자 브라우저에서 실행
  • 반사형(Reflected) — 요청 파라미터가 응답에 그대로 반사되어 실행
  • DOM 기반 — 서버 거치지 않고 클라이언트 DOM 조작으로 실행
🔄 응용 포인트
대응: 출력값 인코딩(핵심)·입력 검증·CSP·HttpOnly 쿠키. "종류만" 쓰면 감점, 대응까지 세트로.
Q. HttpOnly 쿠키가 XSS 피해를 줄이는 원리는?
HttpOnly 속성이 붙은 쿠키는 document.cookie 등 스크립트에서 접근 불가 → XSS로 스크립트가 실행돼도 세션 쿠키 탈취를 차단한다(공격 자체를 막진 못함).
CISO 업무·역할암기27·31 출제
🔑 기출 핵심
두문자 정·위·교·법 — ① 정보보호 관리체계 수립·운영 ② 위험관리·대책 수립 ③ 정보보호 교육·훈련 ④ 법적 준수 여부 점검(취약점 분석·평가·침해대응 포함).
🔄 응용 포인트
정보통신망법상 지정·신고 의무, 일정 규모 이상 겸직 제한 등 제도 요건으로 변형 가능.
접근통제 모델 BLP / Biba응용20·27 계열 단골
🔑 기출 핵심
  • BLP = 기밀성 · No Read Up / No Write Down
  • Biba = 무결성 · No Read Down / No Write Up
🔄 응용 포인트
"No Write Down의 보안적 의의(높은 등급→낮은 등급 정보 유출 경로 차단)"를 명확히 안 쓰면 감점. 규칙 방향을 상황에 대입.
Q. BLP에서 No Write Down이 기밀성을 지키는 이유는?
높은 보안등급 주체가 낮은 등급 객체에 기록하면 기밀 정보가 하위로 흘러 유출된다. 이 하향 정보 경로를 막아 기밀성을 보장한다.
IPSec 모드 / AH·ESP응용28·30 출제
🔑 기출 핵심
  • 전송 모드 = 페이로드만 보호 / 터널 모드 = 원본 IP 전체 + 새 헤더
  • AH = 인증·무결성(암호화 X) / ESP = 암호화 + 인증
🔄 응용 포인트
VPN 시나리오에서 "어떤 모드/프로토콜을 써야 하나"로 판단형 출제. 헤더 구조 그림 설명 요구 가능.
파일 업로드 취약점응용29 출제 · 소스분석형
🔑 기출 핵심
대응: 화이트리스트 확장자·매직넘버 검증·실행권한 제거·난수 파일명·업로드 경로 분리.
🔄 응용 포인트
PHP 소스가 주어지고 취약점명 + 우회기법(이중확장자·널바이트·대소문자) + 성공 조건을 함께 서술(29-14형).
⚠︎ 서술형 감점 방지 3원칙
  • 정의 → 원리/단계 → 보안적 의의·이유까지 3단으로 써라 (정의만=절반)
  • "각각 설명하시오"면 항목 수만큼 번호를 매겨 빠짐없이
  • 대책·대응을 요구하면 근본 대책 1개 + 부가 대책 2개 구조로
03

실무형 시나리오 응용

17–18번 · 택1 · 16점

로그·설정·코드를 읽고 분석하는 순수 응용 영역. 두 문항 중 익숙한 쪽 택1 — 신출 도구(예: 31-17 Fiddler)가 나오면 버리고 아는 쪽을 고르면 됩니다.

Snort 룰 분석·수정응용매 회차급 반복
🔑 기출 핵심
content(탐지 문자열) · offset(검사 시작 위치) · depth(검사 길이) · distance(직전 매치 이후 간격) · nocase(대소문자 무시).
🔄 응용 포인트
"이 룰로 탐지 안 되는 이유 + 수정안"이 단골 → offset:1→0 조정, nocase 추가, 대소문자 우회 대응.
Q. content:"GET";offset:1;depth:3; 의 의미는?
패킷 페이로드의 처음 1바이트를 건너뛰고, 이후 3바이트 안에서 "GET" 문자열을 탐색하라는 의미. 대소문자를 섞은 우회가 의심되면 nocase를 추가한다.
세션·쿠키 / CSRF 시큐어코딩응용31-18 출제
🔑 기출 핵심
세션=서버 저장 / 쿠키=클라이언트(브라우저) 저장. 쿠키 3속성: Secure(HTTPS만)·HttpOnly(스크립트 차단)·SameSite(교차사이트 전송 차단).
🔄 응용 포인트
안전한 코드가 무슨 취약점을 막나 → CSRF. 추가 보완: Referer 검증·CSRF 토큰·SameSite 쿠키.
Q. CSRF 토큰이 공격을 막는 원리는?
요청마다 서버가 발급한 예측 불가한 토큰을 폼에 포함시키고 검증 → 공격자가 위조 요청에 유효 토큰을 넣을 수 없어 차단된다. SameSite 쿠키·Referer 검증과 병행.
SQL Injection / 시큐어코딩응용27·30 출제
🔑 기출 핵심
인증 우회 입력 ' or '1'='1(항상 참). 근본 대응 = Prepared Statement(쿼리 구조 선컴파일, 입력을 데이터로만 처리).
🔄 응용 포인트
취약 쿼리 코드를 주고 취약점 지적 + 안전한 코드로 수정 요구. 입력 검증·최소권한 계정·오류메시지 숨김도 언급.
포트 스캔 / 방화벽 룰셋 판별응용14·계열 반복
🔑 기출 핵심
TCP ACK 스캔 = 포트 오픈 여부가 아니라 방화벽 필터링 여부 판별(RST 응답=미필터링, 무응답/ICMP=필터링).
🔄 응용 포인트
스캔 결과(응답 패턴)를 주고 "어느 포트가 열림/필터링인지" 해석. SYN/FIN/NULL/XMAS 스캔과 구분.
04

출제 임박 레이더 · 서술·실무

순환 주기 × 공백 분석

서술·실무는 직전 회차 반복이 드물고 3~6회 주기로 순환합니다. 그래서 "마지막 출제 이후 공백(gap)이 그 주제의 평균 출제주기를 넘겼는가"로 32회 임박도를 점수화했습니다. 31회에 막 나온 주제(CISO·XSS·SSL·생체·세션쿠키)는 소진으로 후순위입니다.

산식 · 임박도 = 공백(gap) ÷ 평균 출제주기 (주기 최소 3회 바닥값) → 1.0 이상이면 "돌아올 때". 출제 횟수가 많을수록 신뢰도↑. gap은 32 − 마지막 출제 회차. 데이터: 22~31회 서술·실무 76문항.
★ 13~21회 확장 분석 · 옛 회차까지 합치면 이력이 길어져 주기 추정 신뢰도가 올라갑니다(예: DDoS n3→n5, 개인정보조치 n2→n4). 결론은 유지되며, 그 과정에서 정량적 위험평가(SLE·ALE·ROI)가 신규 유력 후보로 확인됐습니다 — 16·19·21회 규칙 출제 후 11회 공백. ⚠︎ 단, 옛날에만 나온 주제(Apache설정·가명정보·포렌식·BLP 등)는 대부분 단답형으로 이동했으므로 서술·실무 후보에서 제외했습니다.

Q. 접속기록을 2년 이상 보관해야 하는 대상은?

🔴2 · 스니핑 탐지응용gap 6
📌 이것만은
전제: NIC를 Promiscuous(무차별) 모드로 두면 모든 패킷 수신. 탐지 4법 —
  • Ping: 위조 MAC + 정상 IP로 ping → 응답하면 무차별 모드
  • ARP: 위조 MAC ARP 요청 → 응답 오면 스니퍼
  • DNS: 스니퍼의 역방향 DNS 조회 발생 감지
  • Decoy(유인): 가짜 계정/암호를 흘려 사용 여부 관찰
대응: 암호화(SSL/SSH)·스위칭 환경.
🔴3 · DDoS·DRDoS 증폭응용gap 5 · 신뢰 높음
📌 이것만은
원리: 출발지 IP를 피해자로 스푸핑 → 반사 서버에 소량 질의 → 대용량 응답이 피해자로 증폭(DNS ANY·NTP monlist·memcached).
  • 증폭 이유: 작은 요청으로 수십~수천 배 트래픽 유발, 출처 은닉
  • 대응: Open Resolver 제거·uRPF(출발지 검증)·RRL(응답률 제한)·ISP 협조 차단
Q. DRDoS가 일반 DDoS보다 위협적인 이유 2가지는?
① 증폭으로 적은 자원에 큰 트래픽을 만든다. ② 반사 구조라 실제 공격자 IP가 드러나지 않아 추적·차단이 어렵다.
🔴4 · 특수권한 SUID/SGID·파일권한응용gap 5
📌 이것만은
  • SetUID 4000(실행 시 소유자 권한) · SetGID 2000 · Sticky 1000(디렉터리 삭제 제한, 예: /tmp)
  • 점검: find / -perm -4000 -type f · 제거: chmod -s 파일
  • 위험: SUID root 실행파일이 침해되면 권한 상승. 불필요한 SUID 제거가 핵심
Q. -rwsr-xr-x 에서 s의 의미와 보안 위험은?
소유자 실행권한 자리의 s = SetUID → 누가 실행하든 파일 소유자(root) 권한으로 동작. 취약한 SUID root 바이너리는 권한 상승 공격의 통로가 되므로 불필요하면 chmod -s로 제거.
🔴5 · 자산 중요도 산정암기gap 4
📌 이것만은
기준 3가지 = 기밀성·무결성·가용성(CIA) 등급 평가 → 자산 중요도 = 세 등급 중 가장 높은 값으로 산정. 정보자산 식별·목록 최신화(ISMS-P 1.2.1).
Q. 기밀성 2·무결성 3·가용성 1이면 자산 중요도 등급은?
3(가장 높은 등급으로 산정). 평균이 아니라 최댓값을 쓰는 이유는 한 축이라도 손상 시 피해가 크기 때문.
🔴6 · DB 접근권한 / 감사로그응용gap 4
📌 이것만은
  • 최소권한 원칙: 업무 필요 최소 권한만 GRANT, 불필요 권한 REVOKE
  • DBA·일반 사용자 권한 분리, 기본계정·기본암호 변경
  • 감사(Audit) 로그 활성화로 접근·변경 추적, 로그 위·변조 방지 별도 보관

★ 정량적 위험평가 (SLE·ALE·ROI)응용·계산gap 11 · 13~21 확장 신규
📌 이것만은 (공식 암기)
  • SLE = AV × EF (단일 손실액 = 자산가치 × 노출계수)
  • ALE = SLE × ARO (연간 손실액 = 단일손실 × 연간발생빈도)
  • 대책 효과 = 감소된 ALE − 대책 운영비용
  • ROI(%) = (ALE − X) / X × 100 (X = 완전 제거에 투입한 비용)
16·19·21회 실무형 계산문항으로 반복. 공식 + 대입값 + 단위를 함께 써야 만점.
Q. AV=100,000 · EF=0.2 · ARO=0.5 일 때 SLE와 ALE는?
SLE = 100,000 × 0.2 = 20,000 · ALE = 20,000 × 0.5 = 10,000. 대책으로 ALE가 30,000 줄고 운영비 17,000이면 효과 = 30,000 − 17,000 = 13,000. (16-15회 실제 값)
